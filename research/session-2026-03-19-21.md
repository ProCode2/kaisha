   - 21 tests, zero leaks
   - Protocol: secrets_sync, secret_update, secret_delete

### kaisha app features
- **Non-blocking UI**: agent runs on background thread, UI stays at 60fps
- **Tool feed**: bottom panel showing tool calls with args, output, diff view
- **Permission system**: inline Allow/Always/Deny buttons in tool feed
- **Steering**: type while agent is busy → "Steer" button redirects agent
- **Secrets panel**: right sidebar, add/edit/delete secrets per box
- **Click-to-copy** on chat bubbles with "Copied!" toast
- **Scroll ownership**: mouse position determines which component gets wheel events
- **System prompt**: lean (~350 tokens) + detailed tool/memory/secrets prompts via @embedFile

## Key bugs fixed
- **Dangling vtable pointers**: ChatScreen returned by value, vtable ptrs pointed to old stack location. Fixed with lazy ensureSetup() and init/setup split.
- **Glob walker stopped on first error**: `walker.next() catch null` broke the while loop. Fixed to `catch continue orelse break`.
- **cwd dangling pointer**: bash.cwd freed on cd, config.cwd stale. Fixed with `cwd_ptr: *const []const u8`.
- **JSON field extraction**: `"command":"value"` vs `"command": "value"` (space after colon). Fixed to try both patterns.
- **Font spacing**: markdown renderer used `spacing * 3` for word gaps. Fixed to use measured font space width.
- **Null-terminated strings for raylib**: `DrawTextEx` needs null-terminated C strings, Zig slices aren't.
- **Permission multi-fire**: keyboard shortcuts fired for all pending entries. Fixed with `accept_perm_input` flag.
- **History resume crash**: `bufPrint` doesn't null-terminate, sentinel slice `[:0]` needs manual null byte.
- **Memory leaks**: SecretStore.set update, SecretProxy.mask intermediate results, secrets tool listNames.
- **API error handling**: non-SSE JSON errors (like "Invalid API Key") parsed and shown instead of empty response.

## Architecture decisions
- **Steering discards pending tool calls** (diverges from pi-mono which delivers steering AFTER tool execution)
- **No MCP** — tool descriptions via @embedFile, not MCP protocol
- **No vector DB for memory** — filesystem grep is sufficient for conversation search
- **Agent self-manages memory** (Letta/MemGPT pattern) — reads/writes .kaisha/memory/ using existing tools
- **<<SECRET:NAME>> syntax** — unique, no conflict with shell $VAR, Jinja {{var}}, or any template language
- **Token efficiency** — treat token waste as a bug, not an optimization
